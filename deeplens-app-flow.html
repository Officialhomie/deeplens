Core Trigger Flows
The two primary ways a user activates DeepLens
FLOW A — HOVER TRIGGER
01
User reads
Mouse moves over dense article, docs, or thread. Normal browsing.
02
Mouse pauses
Cursor rests on a word. Timer starts at 0ms.
t = 0ms
03
Debounce fires
300ms threshold passed. Word extracted via caretRangeFromPoint.
t = 300ms
04
Context built
Sentence, heading, page title, domain category extracted.
t = 310ms
05
Tooltip opens
Shadow DOM tooltip injected below cursor. Skeleton shown.
t = 360ms
06
AI streams in
Tokens appear word-by-word. User reads in real time.
TTFT <800ms
FLOW B — SELECTION TRIGGER
01
User selects text
Click-drag or double-click highlights a word, phrase, or sentence.
02
mouseup fires
Event listener catches mouseup. Selection text extracted via window.getSelection().
03
Validation
Min 3 chars, max 500 chars. Not inside exclusion zone (input, textarea, contenteditable).
04
Tooltip anchors
Tooltip appears at end of selection bounding rect. Auto-flips if near viewport edge.
05
Full phrase context
Richer context injection since phrase > single word. Better AI output.
06
Response streams
User can pin, copy, change mode, or dismiss without losing reading position.
🚫
Exclusion Zones
DeepLens never triggers inside: input fields, textareas, select dropdowns, buttons, contenteditable divs, or any element with role="textbox". Code blocks excluded by default, opt-in via settings.
P0 — hard requirement
⌨️
Keyboard Trigger (Alt+D)
With any text selected, pressing Alt+D immediately triggers Deep mode. User-configurable. Useful on pages where hover is unreliable.
P1
🌐
Domain Blacklist
Users can silence DeepLens on specific domains: banking sites, email, internal tools, password managers. Added via settings popup. Checked before any event listener fires.

📍
Auto-Positioning Logic
Default: 12px below cursor. If bottom overflow → flip above. If right overflow → shift left. If both → bottom-left of cursor. Anchors to selection end rect for multi-word selects.
📌
Pin / Expand Mode
Clicking the pin icon locks the tooltip open and expands it to 480px width and 600px max-height. Pinned tooltips slide in from the right and stay visible even as mouse moves. Dismiss via X only.
🌓
Dark Mode Detection
Priority order: (1) OS prefers-color-scheme, (2) page background luminance check. Tooltip theme switches automatically. All colors defined per theme in Shadow DOM :host scope.

Settings Flow
Everything configurable and how it maps to behavior
🔑
API Key Management
Input field in popup. On save: validated with test ping to Anthropic API. Stored in chrome.storage.local only. Never exposed to page context. Can be cleared (extension deactivates).
P0
🎚
Hover Delay
Slider: 200ms–800ms. Default 300ms. Lower = more responsive (may trigger accidentally on fast reads). Higher = more intentional (better for power-readers). Stored in chrome.storage.local.
P1
🎯
Default Mode
Segmented control: Quick / Deep / Links. Persists across sessions. Overridable per-session by clicking mode toggle in tooltip. Deep is the default default.
P1
🚫
Domain Blacklist
Text input: add domain (e.g. mail.google.com). Extension is completely silent on blacklisted domains — no listeners fire, no tooltip can appear. Default list: notion.so, mail.google.com.
P1
⌨️
Keyboard Shortcut
Configure trigger key combo (default: Alt+D). Works on any selected text to immediately fire Deep mode, bypassing hover. Useful on complex SPAs where hover is erratic.
P1
🌍
Output Language
Dropdown: ISO 639-1 language codes. Default: English. User can receive AI responses in French, Yoruba, Arabic, etc. Injected into system prompt as final instruction.

06
System States
All states the extension can be in and valid transitions between them
IDLE
Extension is active. Event listeners registered. Waiting for user intent. No tooltip visible. Zero resource consumption.
Entry: install · page load · tooltip dismissed
LOADING
Trigger fired. Context extracted. Message sent to service worker. Tooltip shows skeleton state. API call in-flight.
Entry: hover 300ms · mouseup on selection
STREAMING
First token received. Tokens rendering into tooltip body in real time. User is reading response. Mode toggle is live.
Entry: first DEEPLENS_TOKEN received
DONE
message_stop event received. Full response rendered. Streaming cursor hidden. Copy + pin actions available.
Entry: message_stop or done:true token
ERROR
API error, network failure, or invalid key. Error card shown inside tooltip with recovery action. No retry loop — user-initiated only.
Entry: error token or fetch exception
ABORTED
User moved off word before 300ms, pressed Escape, or clicked away mid-stream. AbortController fires. No error shown. Silent return to idle.
Entry: mouseleave · Escape · click-away
VALID STATE TRANSITIONS
IDLE →
→ LOADING (trigger fires)
LOADING →
→ STREAMING (first token)
→ ERROR (API fail)
→ ABORTED (user dismiss)
STREAMING →
→ DONE (message_stop)
→ ERROR (mid-stream fail)
→ LOADING (mode switch)
→ ABORTED (dismiss)
DONE →
→ IDLE (tooltip dismiss)
→ LOADING (mode switch)
ERROR →
→ LOADING (retry button)
→ IDLE (dismiss)
ABORTED →
→ IDLE (only valid transition)
Shadow DOM destroyed
07
Error States and Recovery
Every failure mode with explicit user-facing message and recovery path
NO_API_KEY
No API key configured
Extension activated without API key. Happens when user dismisses onboarding early.
→ Open settings [button]
INVALID_KEY
Invalid API key
HTTP 401 from Anthropic. Key may be expired, wrong, or incorrectly pasted.
→ Check key in settings [button]
RATE_LIMIT
Too many requests
HTTP 429 from Anthropic. Shows retry countdown from Retry-After header. Auto-retries after timer.
→ Retry in [N]s [countdown]
API_OVERLOADED
AI is busy right now
HTTP 529. Anthropic servers overloaded. Auto-retries once after 3s. If fails again, shows manual retry.
→ Retry [button] · auto in 3s
NETWORK_ERROR
No connection
Fetch failed entirely. Network offline or DNS failure. User must fix connectivity themselves.
→ Check your connection
SW_TERMINATED
Connection lost mid-stream
Chrome terminated service worker between messages (MV3 quirk). Stream is interrupted.
→ Tap to retry [button]
